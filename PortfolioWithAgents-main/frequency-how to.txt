Musician / Producer Portfolio — "Frequency"
A single-file, mobile-friendly personal portfolio website. The sample content is for

Maya Rivera, a teen music producer — swap it for your own. Everything lives in one file: index.html.

No accounts, no installs, no build tools. If you can open a file and type, you can ship this.

1. Open it
Make sure index.html is in this folder.
Double-click index.html. It opens in your web browser. That's the whole site.
Tip: To edit it, open the same file in a free code editor like VS Code
(or even Notepad / TextEdit). Browser = view it. Editor = change it.
2. Make it yours (edit the text)
Open index.html in your code editor.
Use Find (Ctrl/Cmd + F) and search for [EDIT]. Every spot you should change is
marked with an [EDIT] comment right next to it.

Change the placeholder text to your own — your name, bio, projects, etc.
Save the file, then refresh the browser tab to see your changes.
That's the core loop: edit → save → refresh.

3. Change the colors
Open the file and find the section near the top that starts with :root {.
Change the --accent color value (a hex code like #c850f0) to any color you like.
Need a code? Search "hex color picker" online.

You can also just click the colored dots in the top navigation bar on the live site to
recolor it instantly — your choice is remembered the next time you open the page.

The moon / sun button in the nav switches between dark and light mode (also remembered).
4. Add your photo
In the hero, find the .avatar-card block and replace the .photo-hint div with:

<img src="me.jpg" style="width:100%;height:100%;object-fit:cover"> (put me.jpg in the same folder).

5. Edit the content blocks
Releases list: each song/project is a .track row in the #releases section. Copy a row to add one; delete a row to remove one.
The animated waveform under the hero text is decorative CSS — leave it or delete the whole .wave block.
6. Use your AI assistant to change anything
You built this with an AI agent — keep using it! Open your project with the agent and ask in

plain English, for example:

"Add a new project card titled ___ with this description."
"Change the headline to say ___."
"Make the whole thing use a blue accent instead."
"The text is too small on my phone — make it bigger on small screens."
Always save and refresh to see the result.

7. Put it online for FREE (so you can share a link)
You have two easy options. Pick one.

Option A — Netlify Drop (easiest, ~2 minutes)
Go to app.netlify.com/drop.
Drag this whole folder (the one containing index.html) onto the page.
Netlify gives you a live URL instantly, like your-site.netlify.app. Share it!
To update later, drag the folder again. (Free Netlify account lets you keep the same link.)
Option B — GitHub Pages (great for your résumé / college apps)
Create a free account at github.com.
Click New repository. Name it your-username.github.io (use your real username).
Make it Public.

Upload index.html (and your photo) using Add file → Upload files, then Commit.
Go to the repo's Settings → Pages. Under "Branch", choose main and /(root), then Save.
Wait ~1 minute. Your site is live at https://your-username.github.io.
To update: upload the changed file again and commit.
Putting the code on GitHub is itself a great thing to show on a college or internship
application — it proves you can build and ship real things.
8. Quick troubleshooting
Fonts look plain / generic: you're probably offline. The site still works; fancy fonts
load from the internet. Reconnect and refresh.

My photo doesn't show: check the file name matches exactly (me.jpg vs Me.JPG) and the
image is in the same folder as index.html.

My changes don't appear: did you save the file and refresh the browser? Try a hard
refresh (Ctrl/Cmd + Shift + R).

I broke something: undo with Ctrl/Cmd + Z, or ask your AI agent to fix the specific part.
